# 前后端同步与按钮 Toast

## 现象

1. 后端任务已结束，前端队列/阶段仍滞后或显示「卡住」。
2. 部分按钮点击后无「已发送」类 Toast 反馈。

## 原因（简要）

- 队列列表与 `next_job_id` 仅靠每 3s 的 `get_queue_state` 轮询更新；`runtime_status` 虽经 WebSocket 实时到达，但未同步拉取队列。
- 任务结束时「阶段」未在状态与 `job_event` 上统一清空。
- 未连接、无位姿、停止入队、审批等路径未统一走 `toast`。

## 改动

- **stateStore**：`applyRuntimeStatus` 在任务 ID 清空或切换时清空 `currentStageName`；新增 `applyQueueStateResponse` 统一解析 `GetQueueState`。
- **wsClient**：`runtime_status` 上 `current_job_id` 变化时立即 `get_queue_state`；`job_event` 终态（SUCCEEDED/FAILED/CANCELLED/REJECTED）时清空阶段并刷新队列。
- **main.js**：轮询间隔 3s→1.5s；各操作增加 `toast`（含未连接/无位姿/清空队列/重新加载模型等）。
- **ApprovalCard**：审批请求与结果增加 `toast.info` / `success` / `warn`。

## 验证建议

- 跑完一次 Pick，观察左侧「队列」「当前执行」与后端是否几乎同步；任务结束阶段应变空。
- 点「停止入队」「审批抓取」「清空」「重新加载模型」等应出现对应 Toast。
