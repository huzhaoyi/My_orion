# 侧抓 grasp/pregrasp IK 与 RViz 对齐

## 问题

`handlePick` 中 `setFromIK(gripper_tcp)` 大量失败，而 RViz 中同一机械臂可规划。根因主要是：

1. **位姿系**：MoveIt `setFromIK` 要求末端位姿在 **kinematic model 参考系**（本机 URDF 根为 `world`）。候选在 `base_link` 下构造，需左乘 `T_world_base` 得到 `T_world_tcp`（`world` 与 `base_link` 为固定关节时通常为单位阵，但显式变换避免与 RViz 内部不一致）。
2. **IK 种子**：原逻辑仅用 PlanningScene 的 `getCurrentState()` 或 `setToDefaultValues()`；RViz 规划使用 **`/joint_states` 当前姿态** 作为种子，KDL 从「全零/默认」常无法收敛到可达目标。

## 修改

- `task_manager.cpp`：`tcp_pose_in_model_frame`、`pick_try_set_from_ik_multiseed`；种子顺序为 **joint_states → PlanningScene → SRDF `ready` → 全默认**；pregrasp 成功时写出 `RobotState` 供碰撞预检。
- `orion_mtc_node.cpp`：订阅 `joint_states`，通过 `setGetLatestJointStateCallback` 提供给 TaskManager。
- `PICK_IK_PER_SEED_TIMEOUT_SEC = 0.15`（每种子一次）。

## 验证

本地 `colcon build --packages-select orion_mtc` 通过；请在 HoloOcean 下再跑 PICK 验证 grasp IK 是否通过并进入规划/执行。
