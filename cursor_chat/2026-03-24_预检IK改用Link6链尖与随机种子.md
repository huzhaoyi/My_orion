# 预检 IK：Link6 链尖与随机种子

## 当日进展

推进 **orion_mtc** 侧抓预检：根据 `orion.srdf` 中 **arm** 组链为 `base_link`→**Link6**、KDL 仅对该链求逆，将期望 **gripper_tcp** 位姿转换为 **Link6** 位姿后再 `setFromIK`（`T_base_link6 = T_base_tcp * T_link6_tcp^{-1}`，固定关节由默认 `RobotState` FK 得到）；同一随机种子下若链尖失败再回退 `tip=gripper_tcp`；多随机种子（6 次）分摊 `ik_timeout_sec`。pregrasp 成功时的 `RobotState` 仍用于后续碰撞预检。

## 完成情况

`colcon build --packages-select orion_mtc` 通过。

## 问题/需求

若仍 NO_IK，需核对缆绳轴向/接近方向与实机 TCP 定义；可调 `feasibility.ik_timeout` 或 `pick_strategy` 多候选。
