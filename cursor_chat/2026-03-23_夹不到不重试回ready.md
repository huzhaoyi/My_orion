# 夹取失败策略：未检测到夹持则不再重试候选，回 ready 结束任务

## 需求

夹不到（传感器未在超时内确认夹紧）时，不要继续尝试下一侧抓候选；直接执行回 `ready`（`retreatToReady`：PTP ready + 张开手），结束本次抓取任务。

## 实现要点

1. **`SolutionExecutor::executePickSolution`**  
   增加可选输出参数 `bool* failed_no_grip_out`：仅在「闭合手后 `wait_for_gripped(true, …)` 超时」时置为 `true`。

2. **`TaskManager::handlePick`**  
   若 `executePickSolution` 失败且 `failed_no_grip` 为真：  
   - 调用 `retreatToReady()`；  
   - 成功：`task_mode_ = IDLE`，`last_error_ = "PICK: 未检测到夹持，已回 ready"`；  
   - 失败：`setStateError("PICK: 未检测到夹持，且回 ready 失败")`；  
   - `return false`（不再 `continue` 下一候选）。

3. **其它执行失败**（轨迹段失败、规划失败等）行为不变：仍可按原逻辑尝试下一候选或最终汇总错误。
