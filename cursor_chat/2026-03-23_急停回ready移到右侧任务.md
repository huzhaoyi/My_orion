# 急停 / 回 ready 移到右侧「任务」并与抓取同路径

## 变更

- **RightPanel**「任务」Tab：在「抓取」卡片下新增卡片「急停与回位」，按钮触发 `orion:emergency-stop`、`orion:go-to-ready`（与 `orion:pick` 一样用 `CustomEvent`）。
- **main.js**：注册上述两个事件，内部与 `orion:open-gripper` 一致：`isConnected` 检查 → `wsClient.callEmergencyStop` / `callGoToReady` → 解析 `values` → 日志 + toast → 成功时 `getQueueState`。
- **TopBar**：移除急停、回 ready 按钮及直接调用 ws 的逻辑，保留停止入队 / 清空 / 重置持物 / 恢复。

## 说明

急停 toast 使用 `toast.warn`（与 `toast.js` 导出一致）。
