# 可配置跳过侧抓预检

## 当日进展

推进 **orion_mtc** `handlePick`：新增参数 **`pick_strategy.skip_side_pick_precheck`**（`MTCConfig::pick_strategy`），在 `task_manager` 中为 true 时不调用 `precheckCableSideGraspCandidate`，直接进入 MTC 规划；启动日志中打印 `skip_side_pick_precheck`，首个候选打一条 WARN 说明已跳过预检。`orion_mtc_params.yaml` 默认置 **true** 便于当前调试，验证完成后可改 **false** 恢复预检。

## 完成情况

`colcon build --packages-select orion_mtc` 通过。

## 问题/需求

跳过预检后规划仍可能因同一几何在 MTC 内失败；恢复安全预检前需解决 KDL/位姿与 TCP 定义问题。
