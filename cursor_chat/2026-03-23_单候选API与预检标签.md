# 单候选 API 与预检标签精简

## 动机

编排层不再持有 `std::vector<CableGraspCandidate>` 与 `candidates[0]`；对外只暴露「一条最优侧抓」。

## 改动

1. **`cable_side_grasp`**  
   - 原 `generateCableSideGrasps` 改为文件内匿名命名空间 `enumerateCableSideGrasps`（不对外声明）。  
   - 对外仅 `generateBestCableSideGrasp` → `std::optional<CableGraspCandidate>`（排序后取首）。

2. **`cable_pick_fail_reason.hpp`**  
   - 增加 `cablePickFailReasonTag()`，预检失败信息一行拼接。

3. **`task_manager::handlePick`**  
   - 使用 `cand_opt` + `precheck(..., 0, ...)`；删除局部 `switch (pre_reason)`。

## 编译

`colcon build --packages-select orion_mtc` 已通过。

## 说明

枚举全部组合再排序的开销仍在 `enumerateCableSideGrasps` 内；若以后要减算力，需在生成阶段早停或改搜索顺序（另议）。
